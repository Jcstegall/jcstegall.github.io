Project 4: Credit Card Spending Analysis- Clustering

Data Introduction

Dataset Overview

We use the Kaggle Credit Card Dataset (CC GENERAL.csv) containing 8,950 anonymized customer records over 6 months.

Link: https://www.kaggle.com/datasets/arjunbhasin2013/ccdata

Key features:

  • BALANCE Current credit-card balance
  • PURCHASES Total purchase amount
  • ONEOFF_PURCHASES Largest single purchase
  • INSTALLMENTS_PURCHASES Amount spent on installments
  • CASH_ADVANCE Cash withdrawn
  • PURCHASES_FREQUENCY How often purchases are made (01)
  • PAYMENTS Total payments made
  • CREDIT_LIMIT Cwustomer credit limit
  • MINIMUM_PAYMENTS Minimum payments made
  • PRC_FULL_PAYMENT Percent of full balance paid
  • TENURE Months as customer (mostly 12)

What is Clustering?

Understanding Clustering

Clustering is an unsupervised machine learning method that groups similar observations into clusters based on patterns in the data. Unlike supervised learning, there are no target labels the algorithm identifies structure on its own.

This project mainly uses K-Means clustering, which works by:

Choosing a number of clusters, k,

Randomly assigning k centroids,

Assigning each data point to the nearest centroid,

Recalculating centroids based on assigned points,

Repeating the process until cluster assignments stabilize.

To choose the best value of k, the project uses both the elbow method and the silhouette score, which compares how similar points are to their own cluster vs. other clusters.

The code also imports hierarchical clustering tools (linkage and dendrogram), which provide an alternative clustering method based on merging observations step-by-step, although the main results come from K-Means.

Data Preprocessing

Processing Steps & Rationale

  • Applied median imputation to handle missing values in selected numeric features
    • Median chosen because it is robust to outliers
    • Prevents model failure due to NaNs in clustering algorithms
  • Created a copy of the dataframe before imputing
    • Avoids Pandas SettingWithCopyWarning and preserves original data
  • Applied StandardScaler to normalize the features
    • Scales each feature to mean = 0, std = 1
    • Essential for distance-based models (e.g., K-Means), where unscaled features distort results
  • If no valid features exist, generates an empty array to prevent downstream errors

This preprocessing ensures the dataset is clean, numerically comparable, and ready for clustering or other ML models.
